Step 1 of 6
Create Your Telegram Bot
Register the bot and get your token. Takes about 2 minutes.
~2 min
1
Open BotFather in Telegram
⌄
a
Open Telegram (app or web)
Use the desktop app, mobile app, or web.telegram.org — any works.
b
Search for @BotFather
In the search bar, type @BotFather — it's an official Telegram bot (blue checkmark). Click on it and press Start.
🔗 Open @BotFather
2
Create a new bot
⌄
a
Send /newbot command
Type and send this message to BotFather:
/newbot
b
Choose a display name
BotFather will ask: "Alright, a new bot. How are we going to call it?"
Type a friendly name, for example:
Family Calendar
c
Choose a username
BotFather will ask for a username. It must be unique and end in bot. Try something like:
family_calendar_42bot
💡
If the name is taken, just add numbers until it works. The username is only used to find the bot — it doesn't matter what it is.
3
Copy your bot token
⌄
a
Find the token in BotFather's message
After creating the bot, BotFather sends a message with your token. It looks like this:
# Your token will look like this: 7284619302:AAHxKqPmWvnLbcXyZ1234abcdef5678ghij
⚠️
Keep this secret! Anyone with this token can control your bot. Never commit it to GitHub. We'll store it safely as an environment variable.
b
Save it somewhere temporarily
Paste it in a notepad — you'll need it in Step 5 (Railway environment variables).
c
Add the bot to your group chat
Create a Telegram group with your 2–4 family members. Then add your new bot to the group: open group info → Add Members → search for your bot username → Add.
💡
The bot works in both direct messages and group chats. For a family setup, add it to a shared group so anyone can send events.
✨
Step 2 of 6

~3 min
1
Go to Google AI Studio
⌄
a
Open Google AI Studio
Click the link below. Sign in with any Google account.
🔗 Open Google AI Studio
b
Click "Create API key"
On the API Keys page, click the blue Create API key button at the top left. Select Create API key in new project.
c
Copy the key
Your key appears — it starts with AIza.... Click the copy icon next to it.
# Looks like this: AIzaSyD8x2Kp7mNqRt3WvYzAbCdEf1GhIjKlMnO
🆓
Free tier: 15 requests/minute, 1 million tokens/day. For 2–4 people sending a few events per day, you'll never hit this limit. No credit card required.

Step 3 of 6
Set Up Google Calendar API
The most steps — but each one is simple. Follow carefully.
~15 min
1
Create a Google Cloud project
⌄
a
Open Google Cloud Console
🔗 console.cloud.google.com
Sign in with your Google account.
b
Create a new project
1. At the top, click the project dropdown (next to "Google Cloud")
2. Click New Project (top right of the popup)
3. Name it calbot
4. Click Create
5. Wait ~10 seconds, then select the new project from the dropdown
2
Enable the Google Calendar API
⌄
a
Go to APIs & Services → Library
In the left sidebar click APIs & Services → Library.
Or use this direct link:
🔗 API Library
b
Search and enable
1. Search for Google Calendar API
2. Click the result
3. Click the big blue Enable button
4. Wait for it to enable (a few seconds)
3
Create a Service Account
⌄
ℹ️
A service account is like a robot Google account that the bot uses to create calendar events. It doesn't need a password — it uses a JSON key file instead.
a
Go to Credentials
Left sidebar → APIs & Services → Credentials
🔗 Credentials page
b
Create the service account
1. Click + Create Credentials → Service Account
2. Service account name: calbot-service
3. Click Create and Continue
4. Skip the optional "Grant access" and "Grant users access" steps
5. Click Done
c
Download the JSON key
1. Back on Credentials page, click on calbot-service account
2. Go to the Keys tab
3. Click Add Key → Create new key
4. Select JSON → click Create
5. A .json file downloads automatically — keep it safe!
⚠️
Never upload this .json file to GitHub! We'll paste its content as an environment variable in Railway instead.
d
Note the service account email
On the Credentials page, you'll see your service account has an email like:
calbot-service@calbot-12345.iam.gserviceaccount.com
Copy it — you'll need it in the next step.
4
Share your Google Calendar with the bot
⌄
ℹ️
You need to share a calendar with your service account, so it has permission to create events. We recommend creating a new shared calendar just for the family bot.
a
Open Google Calendar
🔗 calendar.google.com
b
Create a new shared family calendar (recommended)
1. In the left sidebar, click + next to "Other calendars"
2. Select Create new calendar
3. Name it Family Bot (or anything you like)
4. Click Create calendar
c
Share the calendar with the service account
1. Find your new calendar in the left sidebar
2. Click the three dots (⋮) next to it → Settings and sharing
3. Scroll to "Share with specific people or groups"
4. Click + Add people and groups
5. Enter the service account email: calbot-service@calbot-XXXXX.iam.gserviceaccount.com
6. Set permission to "Make changes to events"
7. Click Send
d
Share the calendar with all family members too
Same settings page → Add each family member's Gmail address with "Make changes to events" permission so everyone can see and edit events.
e
Copy the Calendar ID
Still on the settings page, scroll to "Integrate calendar".
Copy the Calendar ID — it looks like:
abc123xyz@group.calendar.google.com
💻
Step 4 of 6
Get the Code onto GitHub
Put the project files in a GitHub repo so Railway can deploy them.
~10 min
1
Download the project files
⌄
a
Get the files from this conversation
Download all 6 files from the previous message:
bot.py, parser.py, calendar_client.py, requirements.txt, railway.toml, Procfile
💡
Create a folder on your computer called calbot and put all 6 files in it.
b
DO NOT create a .env file
We will not put secrets in any file. We'll add them directly in Railway's environment variable UI. This keeps secrets out of GitHub.
2
Create a GitHub repository
⌄
a
Go to GitHub and create a new repo
🔗 github.com/new
• Name: calbot
• Visibility: Private (important! keeps your code private)
• Click Create repository
b
Create a .gitignore file
In your calbot folder, create a file named .gitignore with this content to prevent accidentally committing secrets:
.env *.json __pycache__/ *.pyc .DS_Store
c
Push your code to GitHub
Open a terminal in your calbot folder and run:
# Initialize git and push git init git add . git commit -m "Initial bot setup" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/calbot.git git push -u origin main
Replace YOUR_USERNAME with your GitHub username.
🚀
Step 5 of 6
Deploy to Railway
Connect your GitHub repo and add your environment variables.
~10 min
1
Create a Railway account & project
⌄
a
Sign up for Railway
🔗 railway.app
Click Login → sign up with your GitHub account (easiest). Railway gives $5/month free credit — more than enough for this bot.
b
Create a new project
1. Click New Project
2. Select Deploy from GitHub repo
3. Authorize Railway to access your GitHub if prompted
4. Select your calbot repository
5. Click Deploy Now
⚠️
The first deploy will fail — that's expected! We haven't added the environment variables yet. Continue to the next step.
2
Add environment variables
⌄
a
Open the Variables tab
In Railway, click on your service → go to the Variables tab.
b
Add all variables one by one
Click "Add Variable" for each row below:
Variable name Value
TELEGRAM_TOKEN
Your bot token from @BotFather
required
GEMINI_API_KEY
Your key from Google AI Studio (starts with AIza...)
required
GOOGLE_SERVICE_ACCOUNT_JSON
Open the downloaded .json file in a text editor, select ALL the content, copy and paste it here as one line
required
CALENDAR_ID
The Calendar ID you copied from Google Calendar settings (e.g. abc123@group.calendar.google.com)
required
MEMBER_EMAILS
Comma-separated Gmail addresses of all members, e.g:
alice@gmail.com,bob@gmail.com
required
TIMEZONE
Your timezone, e.g. Europe/Berlin or Europe/Warsaw
required
c
Save and redeploy
After adding all variables, Railway will automatically redeploy. Watch the Deployments tab — it should show a green Success after ~60 seconds.
🎉

2
Troubleshooting
⌄
🔴
Bot doesn't respond at all
Check Railway → Deployments tab → click the latest deploy → view logs. Look for errors. Most likely TELEGRAM_TOKEN is wrong.
🟡
"Gemini API error"
Your GEMINI_API_KEY is invalid or not copied correctly. Go back to aistudio.google.com and create a fresh key.
🟡
"Google Calendar error: 403"
The service account doesn't have permission on the calendar. Go back to Google Calendar settings and re-share the calendar with the service account email with "Make changes to events" permission.
🟡
Events created but no email invites
Check your spam folder — Google may put the first calendar invite there. Also verify the emails in MEMBER_EMAILS are exactly right.
🔵
View Railway logs anytime
Railway → your project → Logs tab. Every bot interaction is logged here for debugging.
